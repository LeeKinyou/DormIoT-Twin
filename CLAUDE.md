# 任务目标：重构 DormIoT-Twin 物联网数字孪生系统

当前项目是一个包含了 MQTT、Redis、MySQL 和多层网关架构的物联网监控系统。为了适应课程设计展示和单机轻量化部署的需求，我们需要将其彻底重构为**纯 Python 单体架构（Streamlit + 物理特征波形合成算法 + 大模型诊断）**。

请严格按照以下 5 个阶段逐步执行重构。在每完成一个阶段后，请向我确认再进行下一步。

## 阶段一：架构“断舍离”（依赖清理与文件删除）
1. 删除 `docker-compose.yml`，我们不再需要任何外部中间件。
2. 删除 `src/dormiot/gateway/`（包含 MQTT 监听、规则引擎、管道等）。
3. 删除 `src/dormiot/storage/`（包含 Redis 缓存、MySQL 仓储等）。
4. 在包管理配置（如 `pyproject.toml` 或 `uv.lock`）中，移除 `paho-mqtt`, `redis`, `sqlalchemy` 等依赖。
5. 引入新的依赖：`pandas`, `numpy`, `langchain`, `langchain-openai`。

## 阶段二：编写“物理波形合成引擎”（替代原有的高斯噪声）
不要再使用简单的随机数。请在 `src/dormiot/simulation/` 下重写数据生成逻辑（如 `synthesizer.py`）。利用 Numpy 编写一个真实的电表数据合成器：
1. **基准负载（NORMAL）**：使用 `50 + np.random.normal(0, 2)` 模拟日常起伏。配合微小的低频正弦波模拟昼夜节律。
2. **违章热得快/吹风机（ALARM_RESISTOR）**：当触发此状态时，在基准负载上瞬间叠加 1800W，并伴随 `np.random.normal(0, 40)` 的高频剧烈毛刺（模拟纯电阻设备的启动和运行特征）。
3. **违章微波炉（ALARM_MICROWAVE）**：当触发此状态时，在基准负载上叠加一个方波函数：每 5 秒在 +1200W 和 +30W 之间交替直上直下跳动（模拟磁控管的间歇式加热）。
> **要求**：数据引擎需能作为单例对象，提供 `get_next_tick()` 方法，每一秒返回所有宿舍（101-106）当前状态下的真实波形点。

## 阶段三：重构后台数据流（内存驱动）
重写 `app.py` 中的数据流转机制，抛弃原有 Pipeline：
1. 建立一个全局线程安全的 `collections.deque`（例如保留最近 60 秒的历史波形队列）。
2. 在 `app.py` 初始化时，启动一个 Python 守护线程（Daemon Thread）。
3. 该线程每秒调用一次“波形合成引擎”，获取 6 个房间的最新指标（功率、温度等），推入全局队列。

## 阶段四：赛博朋克风大屏 UI 重构
继续使用 Streamlit，但视觉语言需要大幅度升级：
1. **全局风格**：强制使用暗色主题（Dark Theme）。
2. **2.5D 空间拓扑**：在界面中间，用 `st.columns` 建立一个 6 宫格。每个格子代表一个房间，正常时显示幽绿色，当其功率 > 1500W 时，背景色变成醒目的红色。
3. **动态波形图**：在下方使用 Plotly 或 ECharts（开启 `smooth: false` 以保留波形的物理毛刺感），绘制最近 60 秒的实时功率曲线图，折线下方需有颜色渐变填充。通过 `st_autorefresh` 或 `time.sleep` + `st.rerun()` 实现每秒重绘。
4. **控制面板**：侧边栏必须保留“异常注入”按钮，允许手动触发上述的“热得快”或“微波炉”波形特征。

## 阶段五：植入“AI 安全专家”研判模块（杀手锏功能）
抛弃原有的硬编码 `if power > 1500: alarm` 规则。在 `app.py` 中引入大模型能力：
1. 当后台队列检测到某房间功率在 2 秒内飙升超过 1000W 时，截取该房间过去 10 秒的功率数组（Array）。
2. 将这个数组传入 LangChain，调用 OpenAI 兼容格式的 API。
3. **Prompt 设计**：“你是一个物联网数字孪生安防专家。以下是某宿舍过去10秒的真实电能波形数据（瓦特）：{power_array}。该波形呈现[尖峰/方波/持续高频等]特征。请在 50 字内给出专业的安防研判，指出疑似何种违章电器，并给出处理建议。”
4. 将大模型的流式或同步返回结果，实时显示在界面顶部的告警日志卡片中，打字机效果最佳。
